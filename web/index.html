the flywheel

three loops, one machine that compounds.

What you see in the demo — groceries — is one slice of the surface area. Underneath it, Kin is built as three self-reinforcing loops.

note the grocery flow is a small subset — the same loops run for bills, supplies, and services.
growth loop

the product markets itself

Kin manages groceries for thousands of households. Every six hours it looks across all of them, finds something genuinely interesting — egg prices up 40%, Trader Joe's cheaper than Instacart this week — and posts it to social media automatically. That post drives signups. New households generate more data. The content gets sharper. More signups. The loop runs itself.

product loop

every purchase teaches the next one

Someone says "buy me celery." Kin price-sweeps every grocery option it knows. No clear winner? It asks the crowd on social, takes the best answer, buys it, and confirms before checkout. Then it writes everything back — this item, this store, this zip code, this price, this date. Next time anyone nearby asks for celery, Kin already knows where to go. The product improves itself with every request.

monetization loop

revenue scales with work done

Every step in the product loop burns tokens — the price sweep, the social post, the crowd parsing, the database write. Users pay for tokens consumed, with a $20 floor. The more Kin does autonomously, the more revenue grows. Heavy users who want full autonomy naturally spend more without ever feeling nickeled and dimed — they're paying for work actually done.

growth brings households → product serves them → price intelligence grows → content gets smarter → more growth → more tokens burned → revenue scales ↺

The asset in the middle — a price-intelligence database built from real purchases and real crowd responses — is what connects all three loops, and it's the one thing no competitor can copy. You can clone a chatbot. You can't clone thousands of households' worth of what-actually-sold-for-what.

built on

core backend

InsForge

Kin's entire memory is InsForge. Every piece of durable state — roommates and balances, the shared ledger, the grocery list, the household context graph, order jobs mid-flight, scheduled nags — lives in InsForge Postgres, driven over its records API. Receipt photos upload to InsForge object storage and come back as URLs pinned to ledger entries. And because InsForge is agent-native, the agent provisioned its own backend at runtime: tables created remotely by the agent itself, no dashboard, no migrations, no human in the loop. The price-intelligence database at the center of our flywheel — the moat — is an InsForge database.

  • Postgres over RESTMembers, ledger, groceries, facts, jobs, config — every read and write on every message.
  • Object storageReceipt images land in an InsForge bucket and attach to expenses as living records.
  • Agent-provisionedKin created and evolves its own schema through the API — a backend built by its own user.

Nebius AI Studio

all inference

Every token Kin thinks runs on Nebius — three open models in three distinct roles. Qwen3-235B drives the hand-rolled function-calling loop, chaining up to ten tool rounds per message. Qwen3-30B handles the fast paths: relevance classification and structured extraction on every single group-chat message. Qwen2.5-VL-72B reads receipt photos straight into ledger entries. One OpenAI-compatible endpoint, a custom sliding-window rate limiter, zero closed-model dependency.

Nimble

price intelligence

Nimble is the eyes of the product loop. Every "buy me X" triggers a real-time structured price sweep across Instacart, Safeway, Whole Foods, Trader Joe's, Costco, and Target via Nimble's Web API, so Kin routes each order to the cheapest store. Nimble also computes week-over-week price trends — the "egg prices up 40%" signals that seed the growth loop's social posts and get written back into the price-intelligence database.

Kylon

autonomous gtm

The growth loop is a Kylon-powered marketing employee. On a schedule, Kin's GTM agent pulls a Nimble price trend, drafts a post, files it into a Kylon workspace as a draft for approval, and publishes to X through Kylon's tools API — the same draft→approve→publish discipline Kin applies to money, applied to the brand. Marketing that runs while the team sleeps.

You.com

grounded search

When the house asks something the household graph can't answer — a vendor's hours, what a biller charges, whether that plumber is legit — Kin grounds itself with You.com's Search and Research APIs: fast web results for lookups, multi-source research for the deeper questions, cited and current instead of hallucinated.
